## 実装時の作業分担

本システムでは、日付選択、クルー管理、時間帯選択、ポジション配置、保存処理など複数の機能が存在する。そのため、2人で実装を行う場合は、機能のまとまりごとに担当を分けることで、作業範囲を明確にし、実装時の衝突を減らすことができる。

### 担当者A：日付・クルー管理機能

担当者Aは、主に日付選択およびクルー情報の管理に関する機能を担当する。

#### 担当するユースケース

* 日付を選択する
* クルーを登録する
* クルー情報を編集する
* クルー情報を削除する
* クルー一覧を表示する

#### 実装する主な機能

* カレンダー画面の作成
* 日付選択処理
* クルー情報登録・編集画面の作成
* クルー登録処理
* クルー編集処理
* クルー削除処理
* クルー一覧表示処理

#### 主に扱うデータ

* DailySchedule
* Crew

担当者Aは、選択された日付に対して、その日に勤務するクルーの登録・編集・削除が行える状態まで実装する。

---

### 担当者B：時間帯・ポジション管理機能

担当者Bは、主に時間帯選択およびポジション配置に関する機能を担当する。

#### 担当するユースケース

* 時間帯を選択する
* ポジションへ配置する
* ポジション配置を変更する
* ポジショニングを保存する
* 前後の時間帯へ移動する

#### 実装する主な機能

* 時間帯選択画面の作成
* 対象時間帯に出勤しているクルーの抽出処理
* ポジショニング編集画面の作成
* クルーのドラッグ＆ドロップによる配置処理
* 配置変更処理
* ポジショニング情報の保存処理
* 前後の時間帯への移動処理

#### 主に扱うデータ

* TimeSlot
* Position
* PositionAssignment

担当者Bは、選択された時間帯に出勤しているクルーを表示し、各クルーをポジションへ配置して保存できる状態まで実装する。

---

## 共通で決めておく内容

実装を始める前に、以下の内容は2人で事前に決めておく必要がある。

* ファイル構成
* データ構造
* 画面遷移
* 関数名・ルート名
* GitHubでの作業ルール

特に、クルー情報やポジション配置情報のデータ構造は、両方の担当範囲で使用するため、先に統一しておく必要がある。

### データ構造の例

```text
Crew
- id
- name
- start_time
- end_time
- date

TimeSlot
- id
- start_time
- end_time

Position
- id
- name

PositionAssignment
- id
- date
- time_slot_id
- crew_id
- position_id
```

## ファイル構成の例

```text
project/
├─ app.py
├─ models.py
├─ database.py
├─ templates/
│  ├─ calendar.html
│  ├─ crew_list.html
│  ├─ crew_form.html
│  ├─ timeslot_select.html
│  └─ positioning.html
├─ static/
│  ├─ style.css
│  └─ positioning.js
└─ routes/
   ├─ crew_routes.py
   └─ position_routes.py
```

この場合、担当者Aは主に `calendar.html`、`crew_list.html`、`crew_form.html`、`crew_routes.py` を担当し、担当者Bは `timeslot_select.html`、`positioning.html`、`positioning.js`、`position_routes.py` を担当する。

## GitHubでの作業方法

2人で同時に作業する場合は、直接 `main` ブランチを編集せず、機能ごとにブランチを分けて作業する。

```text
main
├─ feature/crew-management
└─ feature/positioning
```

担当者Aは `feature/crew-management` ブランチで作業し、担当者Bは `feature/positioning` ブランチで作業する。

基本的な作業の流れは以下の通りである。

```text
1. main ブランチの最新版を取得する
2. 自分の担当ブランチで作業する
3. 動作確認を行う
4. commit する
5. push する
6. main ブランチへマージする
```

---

## 実装順序

実装は以下の順序で進めると、担当間の依存関係を整理しやすい。

```text
1. 画面の大枠を作成する
2. データ構造を決定する
3. 日付選択機能を実装する
4. クルー登録機能を実装する
5. クルー一覧表示機能を実装する
6. 時間帯選択機能を実装する
7. 出勤中クルーの抽出処理を実装する
8. ポジション配置機能を実装する
9. ポジショニング保存機能を実装する
10. 前後の時間帯移動機能を実装する
```

特に、ポジション配置機能ではクルー情報を利用するため、先にクルー登録・一覧表示機能を実装しておくと、後続の作業が進めやすい。

---

## 分担のまとめ

| 担当   | 主な担当範囲      | 主な機能                                 |
| ---- | ----------- | ------------------------------------ |
| 担当者A | 日付・クルー管理    | 日付選択、クルー登録、クルー編集、クルー削除、クルー一覧表示       |
| 担当者B | 時間帯・ポジション管理 | 時間帯選択、出勤中クルー抽出、ポジション配置、配置変更、保存、時間帯移動 |

このように分担することで、各担当者の作業範囲が明確になり、実装時のファイル競合や処理の重複を減らすことができる。
